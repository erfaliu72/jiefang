# 流程 Review 与测试报告

## 1. Review 依据

依据文件：

- `/Users/liuyuanchang/Desktop/解放-何总/解放车辆流程图3.pdf`

PDF 抽取出的主流程要点：

- 车管录入车辆，车辆信息全员可见。
- 销售发起三类合同：卖车、租车、以租代售。
- 合同审批完成后，必须进入首次付款流程。
- 首次付款由销售发起，以客户付款截图为准。
- 财务查看公司收款，以公司到账截图或银行回单为准。
- 老板审核通过后，结果同步运营和车管。
- 车管只能在付款审核完成后执行出库。
- 后续每一期客户还款走“客户截图 -> 财务回单 -> 平账核销”。
- 未履约后按 T+1 运营催促、T+3 销售催促、T+5 销售发起锁车。
- 锁车必须财务审核历史数据，再由老板确认。
- 退车或续约阶段，销售发起退车，运营、财务、老板审批，车管执行旧车入库。

## 4. 本次发现和处理

已处理问题：

- 合同审批完成后不能直接出库，必须统一进入“待首付款”。
- 卖车合同不再自动进入旧的 `sale_payment` 支付核对流程，改为销售发起首次付款。
- 租赁、以租代售、销售三类合同都已强制首次付款后才能出库。
- 财务审核首次付款前必须上传公司收款回单。
- 首次付款被驳回后，合同状态变为“首付已驳回”，销售可重新发起。
- 租赁和以租代售的首次付款包含首期租金时，系统会自动核销第 1 期客户还款，避免后续重复入账。
- 对账核销、客户还款核销、厂家月供核销均有重复提交保护。
- 退车验收创建后不再提前改变车辆和合同状态，必须审批并由车管确认入库后才结清。

已按本轮需求落地：

- 已新增法务角色、法务账号、法务菜单和合同审批节点。
- 合同审批链已调整为“运营 -> 财务 -> 法务 -> 老板”。
- 所有对账发起统一由销售完成，财务负责上传回单和核销。
- 合同列表已从“合同管理”迁移到“车辆管理”栏目集中查看。
- 合同审批发起和首次支付审批发起已迁移到“审批中心”栏目。

## 6. 已走通业务场景

租赁合同全流程：

```text
车管入库
-> 销售发起租赁合同
-> 运营驳回
-> 销售重新提交
-> 运营/财务/老板审批
-> 状态进入待首付款
-> 车管提前出库被阻止
-> 销售发起首次付款
-> 财务未上传回单时审核被阻止
-> 财务上传回单
-> 财务/老板审核首次付款
-> 状态进入待出库
-> 车管出库
-> 首期租金自动核销
-> 后续账单统一由销售发起对账截图、财务上传回单、财务核销
-> 厂家月供核销
-> 重复核销被阻止
```

销售合同全流程：

```text
车管入库
-> 销售发起卖车合同
-> 运营/财务/老板审批
-> 状态进入待首付款
-> 不再生成旧 sale_payment 流程
-> 销售发起卖车付款
-> 财务上传公司回单并审核
-> 老板确认
-> 状态进入待出库
-> 车管出库
-> 车辆状态变为已售
-> 合同自动结清
```

以租代售全流程：

```text
车管入库
-> 销售发起以租代售合同
-> 运营/财务/老板审批
-> 状态进入待首付款
-> 车管提前出库被阻止
-> 销售发起押金及首次支付审核
-> 财务上传公司回单并审核
-> 老板确认
-> 状态进入待出库
-> 车管出库
-> 车辆状态变为以租代售
```

逾期锁车和退车入库：

```text
逾期账单生成
-> 运营 T+1 催促
-> 销售 T+3 催促
-> 销售 T+5 发起锁车
-> 财务审核
-> 老板确认锁车
-> 车辆状态变为已锁车
-> 销售创建退车验收单
-> 销售提交退车入库审批
-> 运营/财务/老板审批
-> 车管确认旧车入库
-> 车辆回到在库并标记二手车
-> 合同结清
```

## 8. Review 结论

当前系统已经覆盖流程图中的主业务闭环：

- 三类合同都必须先合同审批，再首次付款，再出库。
